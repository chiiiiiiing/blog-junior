### 1. 核心四大模块设计 (Prisma Schema)
请在 backend 的 `schema.prisma` 中显式定义以下四个模型及其实体关系：
- **User (用户/管理员)**: 
  - 包含 id, username, passwordHash, email, role (ADMIN/USER), createdAt。
- **Post (博客文章)**: 
  - 包含 id, title, slug (唯一索引), content, published (Boolean), createdAt, updatedAt。
  - 关联：属于一个 User (Author)，拥有多个 Comment，拥有多个 Like。
- **Comment (评论)**:
  - 包含 id, content, createdAt。
  - 关联：属于一个 Post，属于一个 User (Commenter)。支持级联删除。
- **Like (点赞)**:
  - 包含 id, createdAt。
  - 关联：属于一个 Post，属于一个 User。
  - 联合唯一索引：[userId, postId]，确保一个用户对同一篇文章只能点赞一次。

设计完成后，直接运行 Prisma 迁移命令生成本地 SQLite 数据库文件，并编写 seed 脚本注入至少2个普通用户、1个管理员、3篇带有复杂 Markdown/数学公式的文章以及初始评论和点赞数据。

### 2. 后端 API 接口实现 (Node.js + TS)
请实现并导出以下端点，妥善配置 CORS 跨域中间件，并使用 JWT 中间件保护所有写操作和用户个人相关的接口：
- **用户模块**: `/api/auth/register` (注册), `/api/auth/login` (登录), `/api/auth/me` (获取当前登录状态)。
- **博客模块**: 
  - `GET /api/posts` (文章列表，支持分页)
  - `GET /api/posts/:slug` (文章详情，需连带查询出其所属的评论列表和总点赞数)
  - `POST/PUT/DELETE /api/posts` (只有 ADMIN 角色可以操作)
- **评论模块**: `POST /api/posts/:id/comments` (必须登录发表), `DELETE /api/comments/:id` (允许评论者本人或 ADMIN 删除)。
- **点赞模块**: `POST /api/posts/:id/like` (点赞/取消点赞切换), `GET /api/posts/:id/like-status` (检查当前用户是否已点赞)。

- **首页 (`/`)**: 干净的文章列表展示（含发布时间、分类、标签），支持分页。
- **文章详情页 (`/post/:slug`)**: 完美渲染 Markdown 正文。必须集成高效的 Markdown 解析器（如 remark/rehype），完美支持代码高亮（如 Shiki 或 Prism）以及 LaTeX 数学公式（如 KaTeX）。
- **点赞与评论组件**: 点赞按钮具备即时状态切换与数字动画（使用 Optimistic UI 保证流畅）；评论区支持实时提交、局部刷新和属于当前用户的评论删除功能。
